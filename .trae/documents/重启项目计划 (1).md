## 重启项目计划

### 1. 停止当前服务

检查并停止当前运行的服务：
- 后端服务（端口5000）
- 前端Hexo服务（端口4000）

### 2. 启动后端服务

在`backend`目录下执行：
```bash
npm start
```

### 3. 启动前端服务

在项目根目录下执行：
```bash
npm run server
```

### 4. 验证服务状态

- 检查后端服务日志，确认服务正常启动
- 检查前端服务日志，确认服务正常启动
- 使用浏览器访问前端页面：`http://localhost:4000`
- 测试GitHub登录功能是否正常

### 预期结果

- ✅ 后端服务运行在端口5000
- ✅ 前端服务运行在端口4000
- ✅ 前端页面可以正常访问
- ✅ GitHub登录功能可以正常使用

### 注意事项

- 确保在正确的目录下执行命令
- 检查服务启动日志，确认没有错误
- 如果服务无法启动，检查端口是否被占用

### 技术说明

项目包含两个主要服务：
1. **后端服务**：使用Express框架，处理GitHub登录等API请求
2. **前端服务**：使用Hexo框架，提供静态博客页面

重启项目需要同时重启这两个服务，确保它们都能正常运行。
